rhacoのトレーニングプログラム

なぎ (2010年2月13日 19:39) | コメント(4) | トラックバック(0)
rhacoのトレーニングプログラムでつまずいてたけど、先に進めました。

<?php
include_once dirname(__FILE__)."__init__.php";
Rhaco::import("generic.Urls");
Rhaco::import("database.DbUtil");
Rhaco::import("model.Post");
 
$db = new Dbutil(Post::connection());
$pattern = array(
	//投稿記事の一覧を表示
	"^posts(?:/index)?/?$"=>array("class"=>"generic.Views","method"=>"read","args"=>array(new Post())),
);
$parser = Urls::parser($pattern,$db);
$parser->write();
これを以下のようにする
//投稿記事の一覧を表示
	"^posts(?:/index)?/?$"=>array("class"=>"generic.Views","method"=>"read","args"=>array(new Post()),"default"=>true),
);
結構悩んだけど、できた時はなるほどってなりました。
本来ならば普通に"default"=>trueをつけないでも表示されるのかな...

http://wikihub.org/wiki/rhaco1-doc/100-blog-tutorial/03-model-view-template

Movable Type 5.0 にアップグレード

なぎ (2010年2月 4日 01:08) | コメント(6) | トラックバック(0)
Movable Type 5が出てたからバージョンアップしてみました。
4.2から5.0へのアップデートは何もないサーバーに4.2をインストールするよりもすんなりいきました。

やり方はMovable Type 5 第一回勉強会のサイトを参考にしました。
そこにも書いてあったけど、ウェブサイトのURLが http://uzusionet.com/ に...。

このブログのURLはウェブサイトの utatanebiyori っていうサブドメインの blog ディレクトリ以下ってことになってた。
つまりhttp://utatanebiyori.uzusionet.com/blog/ってこと。
最終的に正しいURLになってたけど、気持ち悪いから直しました。

画像のパスとかどうなってるんだろう...気になります。

ついでにテーマも変更。
http://vicuna.jp/ のデザインを使わせてもらいました。
いろいろと改造していく予定。
